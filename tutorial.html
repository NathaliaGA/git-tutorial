O que é GitHub?

É um sistema open source de controle de versão de código, criado pelo mesmo desenvolvedor do Linux. Com o GIT podemos desenvolver projetos colaborativos, onde outras pessoas podem trabalhar simultaneamente no mesmo código sem riscos de perdermos nada do trabalho.

Por que é importante?

1. Organização - Porque você não vai lembrar o que mudou no projeto, o Git cria uma timeline com tudo que aconteceu desde o início do projeto) 2. Projetos com várias pessoas - Evita o “final_valendo_valendo2_oficial” Ajuda trabalho em equipe ( entender o que foi feito, onde parou, quem fez o que) 3. Segurança - Se algo der errado, pode voltar para versão anterior (ctrl z da vida!)

Comandos básicos

  • ls - LISTAR (ele traz uma lista de tudo o que está naquela pasta - documentos, outras pastas, etc)
  • pwd- Present working directory (onde estou?) Ele traz todo o caminho onde você está (em que pasta e onde essa pasta fica)
  • cd - change directory (use para se locomover entre as pastas)
  • cd ~ - volta liara a raiz
  • cd . - volta uma liasta acima
  • cd nome-da-liasta - liara entrar em uma pasta (você precisa conseguir enxergar ela quando listar os arquivos)

Comandos que mais vamos usar

  • git init - avisa o git para ficar de olho na pasta
  • git status - nos mostra o que foi modificado, adicionado, ou deletado da pasta
  • git add nome-do-arquivo.html - vamos colocar na caixinha essa versão do arquivo
  • git add . - vamos colocar na caixinha todos os arquivos
  • git commit -m “inclui botão vermelho” - vamos colocar a etiqueta na caixinha
  • git log - mostra a timeline do nosso projeto com todos os commits que fizemos
  • git checkout -- arquivo.txt- para voltar para a versão do último commit
  • git branch nome-do-branch - para criar um branch (um espacinho separado para trabalharmos, e depois que estiver tudo lindo, juntamos na nossa master novamente)
  • git checkout nome-do-branch - para trocarmos de branch (antes nós só criamos o branch!)
  • git merge nome-do-branch - para juntar o nosso branch com nosso projeto principal

Usando o git no terminal

Como saber se o GIT está instalado?

Digite git na linha de comando, e várias instruções e sugestões do git devem aparecer.

Vamos criar nosso repositório

Profile > Repositories > New

Preencher os dados e criar (repare: repositório segue o padrão do Github, que será o endereço do Github + seu usuário + nome do repositório)

  • git config --global user.name “Exemplo1"
  • git config --global user.email “exemplo1@gmail.com"

Vamos colocar nosso projeto no github via terminal

Comandos que mais vamos usar:

  • git remote add origin https://github.com/apdsrocha/try_git.git - para indicar em qual repositório vamos colocar nosso projeto
  • git push origin master - “empurrar” nosso projeto para a nuvem
  • git pull origin master - “puxar” o projeto para nossa maquina, carregar as mudanças dos colegas que alteraram
  • git clone endereco-do-projeto-la-no-github.git - copia todos os arquivos do projeto para a nossa máquina
